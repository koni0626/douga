# ADR-0010: 静止区間を1回だけ描画して再利用する

- Status: Accepted
- Date: 2026-07-20

## Context

現在の書き出しは、視覚状態が変化しない区間でも出力フレームごとにHeadless ChromiumでSVGを描画し、PlaywrightでPNGを取得している。Dougaは静止画像とテロップ主体であり、長尺動画では同一画面の再描画が書き出し時間の主要因になる。

実時間でWebプレビューを再生して録画する方式は、素材読込やブラウザスケジューリングの遅延により、画像切替の暗転や音声・テロップのずれを生じるため採用しない。

## Decision

Project Documentから決定的な出力フレーム列とRender Planを生成する。時間に依存しないことを構造的に証明できる区間は先頭フレームを1回だけ描画して再利用し、アニメーション、カメラ、フェード、文字送りなどの区間は従来どおり各フレームを描画する。

初期実装では、再利用したPNGを既存のFFmpeg `image2pipe`へ必要回数送る。FFconcat duration manifestは、正確なフレーム数と音声同期の技術検証後に追加する。

未知のレイヤー型または効果は静止と推測せず、該当区間を動的として扱う。最適化は設定で無効化でき、現行の全フレーム描画へ戻せるようにする。

詳細は[静止区間再利用型レンダリング設計](../static-segment-rendering-design.md)に定義する。

## Consequences

- 静止画主体の動画ではChromium描画とPNG生成回数を大幅に削減できる。
- プレビューと書き出しは引き続き同じScene Rendererを使用する。
- Render PlanがProject Documentの時間依存機能をすべて把握する必要がある。
- 新しい視覚効果は、静止判定を登録するまで毎フレーム描画される。
- 初期段階ではFFmpegが同じPNGを複数回展開するコストが残る。
- 画像の100枚バッファリングと複数ブラウザ並列化は導入しない。
