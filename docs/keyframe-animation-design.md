# キーフレームアニメーション設計

## 1. 目的

オブジェクトごとの位置、サイズ、回転、透明度、反転、色を時間に沿って変化させる。通常編集の簡潔さを保つため、プロパティ別トラックは常時表示せず、オブジェクトの状態を1つのキーフレームとして記録する。

## 2. UI

- オブジェクトを右クリックし、「アニメーション」または「エフェクト」からプリセットを選択する。
- プリセット選択時に、現在時刻と指定した長さから必要なキーフレームを自動生成する。
- アニメーション設定後に別の時刻でキャンバス操作または数値変更を行うと、キーフレームを自動追加する。
- キーフレームはオブジェクトバー上のダイヤで表示する。
- ダイヤを選択すると、小さなポップアップで補間方法、現在位置への複製、削除を操作できる。
- ロック中のオブジェクトにはキーフレームを追加しない。
- 現在の動画終端を越えるキーフレームを生成する場合、動画時間を次の50ミリ秒単位まで自動延長する。

初期プリセット:

- アニメーション: スライド4方向、ズームイン、ポップ、バウンド、シェイク、回転、パルス、浮遊
- エフェクト: フェードイン、フェードアウト、点滅、フラッシュ

フェードインと登場系アニメーションは現在時刻で完了する。フェードアウトと強調系アニメーションは現在時刻から開始する。長さは0.3秒、0.6秒、1.0秒、2.0秒から選択する。

## 3. データモデル

各レイヤーは省略可能な `keyframes` 配列を持つ。既存プロジェクトには同フィールドがないため、従来どおり静止状態として扱う。

各キーフレームは次を保持する。

- IDと時刻（ミリ秒）
- 直前のキーフレームから到達する際に使用する補間方法
- x、y、width、height、rotation、opacity
- 左右反転、上下反転
- 図形の色、テキスト色、フォントサイズ（対象レイヤーのみ）

同一レイヤー・同一時刻の記録は既存キーフレームを置き換える。配列は時刻順に保存する。

## 4. 補間

初期対応は次の6種類とする。

- linear: 一定速度
- ease_in_out: なめらか
- ease_in: ゆっくり開始
- ease_out: ゆっくり停止
- bounce: 弾む
- step: 瞬時に切り替え

数値は補間し、色は16進RGB同士の場合に各チャンネルを補間する。反転は次のキーフレーム到達時に切り替える。最初のキーフレーム以前は最初の状態、最後のキーフレーム以後は最後の状態を維持する。

## 5. 描画と出力

補間処理は `scene-renderer` に集約する。Reactプレビューとヘッドレス動画出力の双方が同じ処理を利用し、フレーム差異を防ぐ。

## 6. 将来拡張

ぼかし、明るさ、モノクロなどの画像フィルターと、継続的なループ指定を追加候補とする。プリセットは専用の描画処理を追加せず、複数のキーフレームを自動生成するテンプレートとして拡張する。必要になった場合のみプロパティ別の詳細トラックを追加する。
